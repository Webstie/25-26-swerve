## Architecture Overview

This is a WPILib 2026 command-based FRC robot using a swerve drivetrain (Phoenix 6 / CTRE).

**Entry point**: `RobotContainer.java` ‚Äî instantiates all subsystems, registers PathPlanner named commands, configures button bindings, and sets up the auto chooser.

**Robot flow** (`Robot.java`):
- `autonomousInit`: calls `getAutoInitCommand()` (resets odometry) then `getAutonomousCommand()` (PathPlanner auto)
- `teleopPeriodic`: optionally fuses vision measurements into odometry via `addMeasurements()`

### Subsystems (`src/main/java/frc/robot/subsystems/`)

| Subsystem | Description |
|---|---|
| `CommandSwerveDrivetrain` | Phoenix 6 swerve, extends TunerSwerveDrivetrain. Integrates PathPlanner AutoBuilder and vision pose estimation. |
| `Launcher` | Friction wheel shooter (3√ó TalonFX) + feeder (TalonFX) + angle motor (SparkMax brushed). VelocityTorqueCurrentFOC for wheels, MotionMagic for angle. |
| `Intake` | Roller (TalonFX/CANivore) + pitch (TalonFX/rio) + support (TalonFX/CANivore). MotionMagic for pitch, CANcoder feedback. |
| `Transport` | Single TalonFX feeder/conveyor on rio bus. |
| `Climber` | Single TalonFX on CANivore, MotionMagic position control, brake neutral. |
| `Vision` | PhotonVision multi-camera, MULTI_TAG_PNP_ON_COPROCESSOR. Returns `VisionMeasurement` records. |
| `CANdleSystem` | Dual CANdle LED controllers (IDs 1 & 2), state-driven animations tied to `RobotState.State`. |

### Commands (`src/main/java/frc/robot/commands/`)

- **`ShootingCommand`** ‚Äî Sequential shoot sequence: warmup friction wheels ‚Üí feed + transport + intake swing. Has dynamic variant using vision to calculate pitch from distance.
- **`MagicSequencingCommand`** ‚Äî Factory class for auto-score sequences at fixed field positions (0‚Äì5). Handles blue/red alliance mirroring.
- **`MoveWhileAimCommand`** / **`AutoMoveWhileAimCommand`** ‚Äî Drive while aiming at target, used for mobile shooting.
- **`OuttakeCommand`** ‚Äî Reverses intake + transport.

### Constants & Configuration

- **`Constants.java`** ‚Äî All tuning values: PID gains, motor IDs, field dimensions, LED configs, `RobotState.State` enum.
- **`generated/TunerConstants.java`** ‚Äî Auto-generated by Phoenix Tuner X. Do not edit manually. Contains swerve module geometry, gear ratios, motor IDs.

### CAN Bus Layout

- **CANivore** (high-speed): Drivetrain modules, launcher friction wheels, intake roller/support, climber
- **Rio CAN** (standard): Intake pitch motor, transport motor, CANdles

### PathPlanner Autos

Auto files in `src/main/deploy/pathplanner/autos/` (BlueAuto1‚Äì7). Named commands registered in `RobotContainer` map to composed WPILib commands:
- `WarmUp_Auto_Far/Near`, `Shoot_Auto_Blue_*` ‚Äî Shooting at fixed field positions
- `Intake_Auto` ‚Äî Deploy and run intake
- `Climb_Auto`, `Climb_UP/DOWN` ‚Äî Climbing sequences
- `AutoMoveWhileAim` ‚Äî Mobile aiming

Red alliance autos are mirror-adapted from Blue via PathPlanner's alliance flip feature.

### Vendor Libraries

- **Phoenix 6** (`Phoenix6-26.1.1.json`) ‚Äî TalonFX motor control, CANcoder, CANivore
- **Phoenix 5** (`Phoenix5-5.36.0.json`) ‚Äî Legacy support
- **REVLib** (`REVLib.json`) ‚Äî SparkMax (intake angle motor)
- **PathPlannerLib** (`PathplannerLib-2026.1.2.json`) ‚Äî Autonomous path following
- **PhotonLib** (`photonlib.json`) ‚Äî Vision pose estimation
- **AdvantageKit** (`AdvantageKit.json`) ‚Äî Logging

## Key Development Notes

- **Alliance adaptation**: Blue alliance is the primary frame. `isRedAlliance()` flips targets and field positions where needed. PathPlanner handles path mirroring automatically.
- **Vision fusion toggle**: `isVisionPoseFusion` boolean in `RobotContainer` controls whether vision measurements are fused during teleop. Toggled with Driver B button.
- **Slow mode**: Driver Start toggles 25% speed cap; default drive is 75% of max.
- **Lombok**: Used for `@Getter`, `@Value`, etc. ‚Äî required for compilation.
- **Simulation**: `CommandSwerveDrivetrain` uses a `Notifier` at 5ms for sim physics; no special sim setup needed beyond `simulateJava`.
